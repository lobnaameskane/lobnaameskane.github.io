Project details

Having reached the end of the first year in the Data Engineer sector within the INPT, an end-of-year project is requested to be accomplished, Our choice relates to design and produce a software product: this is a desktop application in Java. We have chosen the tools in a way consistent with our work philosophy and in harmony with the knowledge acquired during this first year. Thus, the search and sorting then the classification according to the importance of the data collected from the files, is sometimes painful especially when it comes to opening them and search their content manually. This problem is quite common among companies that have Txt, PDF, doc, etc. files as database support The automation of research through algorithms that collect data and sort them according to criteria of importance, for example the date of last modification, then displays the results in a more visible and meaningful form. That automation forms the ideal solution. In addition, we can ensure the speed and ease of implementation thanks to a simplified graphic interface making it possible to adapt the application to the company profile. The application we are developing allows you to: - Fetch user search. - Read and collect the content of files (extension: Txt, PDF, doc and docx). - Sort and classify the files according to criteria (which we will detail later). - Show files more prominently in a table. - Propose a history file that contains the search result, as well as secondary data (number of occurrences, link to the file, etc.). Finally, this project entitled Dfinder revolves around three main points: - The first point: to propose relevant research methods so that the search result is more meaningful adaptive to the user's need. - The second point: to offer a customizable graphical interface (personalization of sorting, backup directory, and more…). - The third point: propose the possibility of saving the result of the research (contains more research details) as a txt file
